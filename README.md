# Day 2 – API Automation (PyTest)

## Overview

This project demonstrates **API automation** using Python, PyTest, and Requests.  
It focuses on:

- GET requests to OpenWeather API
- JSON Schema validation (contract testing)
- Temperature conversion (Kelvin → Celsius)
- Parametrized tests for multiple cities
- Negative test for invalid API key
- HTML report generation
- CI/CD integration via GitHub Actions

## Installation
1. Clone the repository:

```bash
git clone <repo-url>
cd api_automation

2. Create a Python virtual environment:
python -m venv venv
source venv/bin/activate   # Linux/macOS
venv\Scripts\activate      # Windows

3. Install dependencies:
pip install -r requirements.txt

Running Tests Locally:
pytest -v -s tests/ --html=reports/weather_report.html

Parametrized Tests
Multiple cities supported:
@pytest.mark.parametrize("lat, lon, city", [
    (24.7136, 46.6753, "Riyadh"),
    (21.3891, 39.8579, "Makkah al Mukarramah"),
    (21.4858, 39.1925, "Jeddah")
])


*** CI/CD Integration ***
GitHub Actions workflow runs tests automatically on:
Push to main
Pull requests

HTML report uploaded as artifact for download:
- name: Upload HTML report
  uses: actions/upload-artifact@v3
  with:
    name: weather-report
    path: reports/weather_report.html
